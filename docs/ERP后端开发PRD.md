# ERP 后端开发 PRD

## 1. 文档目标

本 PRD 用于指导后续使用 Python 和 MySQL 开发完整后端程序。当前仓库已有 React + Vite 前端，页面模块和交互基本成型，后端建设目标是把前端静态状态数据替换为真实接口和数据库数据，前端界面暂不做结构性改版。

## 2. 信息来源

- 前端程序：`src/App.tsx`、`src/types.ts`、`src/components/*Screen.tsx`
- 主数据：`docs/2026年市场部业务台账.xlsx`
- 字段说明：`docs/台账字段说明0512.xlsx`
- 主台账工作表：`供应链公司业务2026`
- 主台账规模：87 列，491 条有效业务明细行

## 3. 产品定位

建设一个面向供应链业务台账的轻量 ERP 管理系统，覆盖订单、采购、交付、销售、收票、付款、开票、回款、应收应付、利润和系统维护。系统以 Excel 台账为初始数据源，后续改为数据库持续维护，并保留原始导入留痕，便于核对财务口径。

## 4. 用户角色

| 角色 | 主要职责 | 权限范围 |
| --- | --- | --- |
| 管理员 | 系统维护、数据导入、用户管理、备份恢复 | 全部模块 |
| 市场/业务人员 | 维护订单、交付、销售合同、开票、回款 | 业务台账、订单、销售 |
| 采购人员 | 维护采购厂商、采购合同、收票、入库、付款计划 | 采购、应付 |
| 财务人员 | 校验发票、付款入账、应收应付核对 | 财务字段、报表 |
| 只读查看人员 | 查看首页、台账和报表 | 查询与导出 |

## 5. 范围边界

### 本期包含

- 登录认证、用户角色、操作日志。
- Excel 台账导入，包含批次、原始行、校验结果。
- 首页仪表盘真实数据接口。
- 台账管理：按项目编号、部门、客户经理、客户单位、订单号、状态、日期范围查询。
- 订单详情：订单基础信息、订单行、金额、数量、交付进度。
- 采购详情：采购厂商、采购金额、采购合同、收票、入库、付款、应付账款。
- 销售详情：销售合同、开票、回款、应收账款。
- 系统管理：操作日志、数据库备份记录、导入任务记录。
- 报表导出：按查询条件导出台账、订单、采购、销售明细。

### 本期不包含

- 独立客户档案 CRM。
- 独立供应商档案 SRM。
- 复杂审批流引擎。
- 前端界面重设计。
- 多组织复杂权限模型。

客户单位、采购厂商等信息保留在业务单据中，并可后续按需要沉淀为主数据。

## 7. 核心业务流程

1. 数据初始化：管理员上传 Excel，系统创建导入批次，保存原始行，解析为订单、采购、交付、销售、财务记录。
2. 订单建档：以项目编号、订单号和订单行作为核心业务主线（项目编号唯一，但是一个项目编号可以对应多个订单号），记录客户、部门、客户经理、业务类型、统计类别、货物、数量、单价、订单价值。
3. 采购跟踪：按订单行记录采购厂商、采购金额，并维护采购合同、收票、入库、付款和应付账款。
4. 交付跟踪：记录交付日期、交付数量、交付金额、待交付数量和待交付金额。
5. 销售跟踪：维护销售合同、合同价值、开票单据、发票金额、回款金额、应收款和关闭状态。
6. 财务核对：系统根据收票、付款、开票、回款数据计算应付账款、应收款、毛利润、毛利率。
7. 首页汇总：按部门、业务类型、月份、关闭状态聚合订单金额、采购金额、毛利、应收、应付和回款。

## 8. 功能需求

### 8.1 登录与权限

- 支持用户名密码登录。
- 后端返回 JWT 或等价会话令牌。
- API 按角色限制写操作。
- 所有增删改、导入、导出、备份、恢复动作写入操作日志。

### 8.2 Excel 导入

- 支持上传 `2026年市场部业务台账.xlsx` 同结构文件。
- 第 3 行为字段表头，第 5 行开始为业务数据。
- 导入前校验表头，识别重复列名，并按列位映射到不同业务字段。
- 导入时保存原始 JSON 行，便于回溯。
- 支持预览导入结果、失败原因、成功行数、失败行数。
- 支持按批次回滚本批次生成的业务数据。

### 8.3 台账管理

- 查询条件：项目编号、部门、客户经理、客户单位、订单号、订单状态、订单日期范围。
- 列表字段：项目编号、客户单位、项目名称、订单金额、采购金额、回款合计、应收款、毛利润。
- 新增台账时可创建项目、订单头和一条基础订单行。
- 导出当前查询结果。

### 8.4 订单详情

- 查询条件：项目编号、订单号、订单日期、业务类型、客户单位、订单日期范围。
- 列表字段：项目编号、订单号、订单日期、货物名称、数量、订单价值、交付数量。
- 支持新增订单行。
- 数量、单价、订单价值按数据库保存值返回，后端可在写入时校验公式一致性。

### 8.5 采购详情

- 查询条件：项目编号、订单号、客户经理、部门。
- 列表字段：项目编号、订单号、客户经理、采购合同号、合同金额、收票金额、付款金额。
- 支持新增采购记录。
- 支持维护采购合同、收票、入库、财务入账、一期付款、二期等多期付款。

### 8.6 销售详情

- 查询条件：项目编号、订单号、客户经理、部门。
- 列表字段：项目编号、订单号、客户经理、销售合同号、合同签订日期、合同价值、开票金额。
- 支持新增销售合同。
- 支持维护开票和一/二期等多期回款。

### 8.7 首页仪表盘

- KPI：订单总金额、毛利润、订单总数、应收账款、应付账款、已关闭订单数。
- 趋势：按月统计订单金额与毛利润。
- 部门排行：按订单金额或毛利润排序。
- 最近操作日志：展示最新 4 条。
- 后端接口返回结构尽量贴合现有前端展示，减少前端改动。

### 8.8 系统管理

- 操作日志分页查询。
- 备份文件列表。
- 手动创建数据库备份记录。
- 记录备份文件名、大小、时间、状态、存储路径。

## 9. 数据口径

| 指标 | 计算口径 |
| --- | --- |
| 订单价值 | 订单行数量 * 销售单价，优先使用 Excel 已给值 |
| 采购金额 | 订单行数量 * 采购单价，优先使用 Excel 已给值 |
| 交付价值 | 交付数量 * 销售单价 |
| 待交付金额 | 订单价值 - 交付价值 |
| 应付账款 | 采购金额 - 已付款金额 |
| 不含税毛利润 | 不含税收入 - 不含税成本 |
| 毛利润 | 订单价值 - 采购金额，并结合税金/退税口径校正 |
| 回款合计 | 一期回款金额 + 二期回款+三期等多期回款总金额 |
| 应收款 | 订单价值 - 回款合计 |
| 是否关闭 | Excel 当前为“关闭”或按应收款为 0 判断关闭 |

## 10. 技术方案

| 层级 | 建议技术 |
| --- | --- |
| 后端框架 | FastAPI |
| ORM | SQLAlchemy 2.x |
| 数据库 | MySQL 8.0 |
| 迁移 | Alembic |
| 数据校验 | Pydantic |
| Excel 解析 | openpyxl |
| 鉴权 | JWT + passlib/bcrypt |
| 测试 | pytest |

## 11. 非功能需求

- 金额字段使用 `DECIMAL(18,2)`，比例字段使用 `DECIMAL(10,6)`。
- 所有业务表包含 `created_at`、`updated_at`、`deleted_at`。
- 列表接口默认分页，默认每页 20 条，最大 200 条。
- 常用筛选字段建立索引。
- 导入必须幂等：同批次不可重复导入，同业务键重复时按策略更新或跳过。
- API 返回统一错误结构，包含错误码、错误信息和字段级错误详情。
- 保留 Excel 原始字段，方便与财务台账核对。

## 12. 验收标准

- 能从现有 Excel 导入 491 条业务明细。
- 首页、台账、订单、采购、销售、系统管理页面均可从 API 获取真实数据。
- 台账列表的订单金额、采购金额、回款合计、应收款、毛利润与 Excel 汇总口径一致。
- 支持按前端已有筛选条件查询和分页。
- 支持新增订单、采购、销售记录并写入操作日志。
- 支持导出查询结果。
- 后端有基础单元测试和导入解析测试。
